# 版本发布自动化脚本使用说明

## bump-and-release.sh

自动化版本发布脚本，一键完成「更新版本号 -> 编译打包 -> 发布到本地服务器」的完整流程。

### 功能特点

✨ **全自动流程**
- 自动更新 package.json 中的版本号
- 使用快速编译模式（跳过 typecheck）
- 自动发布到本地 Nginx 服务器
- 失败时自动回滚版本号

🎯 **灵活的版本控制**
- 支持语义化版本递增（patch/minor/major）
- 支持手动指定具体版本号
- 更新前后对比显示

⚡ **快速构建**
- 使用 VITE_FAST_BUILD 模式
- 跳过类型检查
- 禁用 sourcemap 和 minify
- 大幅减少构建时间

### 使用方法

#### 1. 递增补丁版本（最常用）

```bash
./bump-and-release.sh patch
```

- 示例：1.0.9 -> 1.0.10
- 适用场景：Bug 修复、小改进

#### 2. 递增次版本

```bash
./bump-and-release.sh minor
```

- 示例：1.0.9 -> 1.1.0
- 适用场景：新功能添加

#### 3. 递增主版本

```bash
./bump-and-release.sh major
```

- 示例：1.0.9 -> 2.0.0
- 适用场景：重大更新、API 变更

#### 4. 指定具体版本号

```bash
./bump-and-release.sh 1.2.3
```

- 适用场景：需要跳版本或修正版本号

### 执行流程

1. **更新版本号**
   - 读取当前版本
   - 使用 `npm version` 命令更新
   - 显示版本变化

2. **编译打包**
   - 运行 `npm run build:win:local:fast`
   - 生成 Windows 安装包
   - 自动调用 publish-local.sh

3. **发布到本地**
   - 复制文件到 Nginx 目录
   - 清理旧版本文件
   - 准备自动更新测试

### 测试更新

脚本成功后，按以下步骤测试：

1. 启动应用程序
2. 点击左侧菜单底部的「检查更新」按钮
3. 查看是否检测到新版本
4. 点击「立即下载」
5. 下载完成后点击「重启安装」
6. 验证应用是否更新到新版本

### 注意事项

⚠️ **构建前提条件**
- 必须在 macOS 系统上运行
- 需要已安装 Node.js 和 npm
- 需要已完成 `npm install`

⚠️ **本地服务器配置**
- Nginx 需要正在运行
- 目标目录：`/opt/homebrew/var/www/svn/auto-updates`
- 访问地址：`http://localhost:8888/auto-updates`

⚠️ **版本控制建议**
- 使用语义化版本规范（Semantic Versioning）
- 生产环境建议先手动运行 `npm run typecheck` 验证
- 可以结合 Git 使用，手动打 tag

### 快捷命令对比

| 操作 | 传统方式 | 新方式 |
|------|---------|--------|
| 修改版本 | 手动编辑 package.json | ✅ 自动完成 |
| 编译打包 | `npm run build:win` | ✅ 自动完成 |
| 发布本地 | `./publish-local.sh` | ✅ 自动完成 |
| 总耗时 | ~5 分钟 + 手动操作 | ~2 分钟全自动 |

### 错误处理

如果编译失败，脚本会：
1. 显示错误信息
2. 自动回滚版本号到旧版本
3. 退出并返回错误代码

### 相关脚本

- `build-release.sh` - 完整的多平台发布脚本（耗时较长）
- `publish-local.sh` - 仅发布已构建的文件到本地服务器
- `build:win:local:fast` - npm 脚本，快速构建 + 发布
